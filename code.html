The code for STRINGREVERSION

                    
                        // Get the string from the page. 
                        // CONTROLLER FUNCTION 
                        function getValue() { 

                            document.querySelector("#alert").classList.add("invisible"); 

                            let userString = document.querySelector("#userString").value; 
                            let revString = reverseString(userString); 
                            displayString(revString); 

                        }

                        // Reverse the string. 
                        // LOGIC FUNCTION 
                        function reverseString(userString) {

                            let revString = ""; 

                            // reverse a string using a for loop 
                            for(let i = userString.length - 1; i >= 0; i--) {

                                revString += userString[i];   

                            } 

                            return revString; 
    
                        }

                        // Display the reversed string to the user. 
                        // VIEW FUNCTION 
                        function displayString(revString) {

                            // Write message to the page.
                            document.querySelector("#msg").innerHTML = `Your string reversed is: ${revString}`; 
    
                            // Show the alert box. 
                            document.querySelector("#alert").classList.remove("invisible"); 

                        }

The Code is structured in three functions: a controller function, a logic function, and a view function.

getValue the Controller Function

getValue's local variables:

  • userString
  • revString

getValue is the controller function of this short application. When invoked, it calls upon the logic and view functions to achieve the goal of the coding project: to reverse the user inputed string. This function is attached to #btnSubmit (code not stated in this script), an HTML button element within the App page. Within the App page, there's an internal script that essentially wires getValue's functionality to the HTML button element as such: document.querySelector("#btnSubmit").addEventListener("click", getValue);

The first line of code scoped inside of getValue accesses the classList property (a DOMTokenList) of #alert and calls upon its add method to push a string value ("invisible") into it. Note that document.querySelector("#alert") references an HTML element which when classed via Bootstrap with "invisible", it prioritizes a CSS regulation to said element equivalent to visibility: hidden !important.

userString

The next line of code declares and initializes userString to the value property of #userString which is referenced by document.querySelector("#userString"). #userString is the HTML textual type input element which takes string input.

revString

In the next couple lines of code, revString is declared and initialized to the return value of reverseString passing userString as an argument returning the reversed string then displays it by calling displayString passing revString as the argument in the final line of code of getValue.

reverseString the Logic Function

reverseString's local variables:

  • revString
reverseString's argument list:
  • userString

reverseString is the logic function of this short application. When invoked, it takes the declared argument, userString, and initializes it to the value of the string inputed by the user (userString aforementioned in the controller function), and is utilized to manipulate the return value of this function (revString).

revString

Before the value of this variable is returned by this (logic) function, it is first declared and initialized to an empty string ("") in the first line of code. It is then manipulated with a for loop to spell out the user inputed string in reverse. The initial expression of the loop declares and initializes variable i to the difference between the length property of userString and 1. The condition expression then evaluates whether i is greater than or equivalent to 0. If the condition expression returns true, the code block executes appending the last character of userString to the empty string initialized by revString (revString += userString[i], much like that of an array although strings are not technically arrays). Code scoped by the for loop executes consecutively until its condition expression returns false, then finally, the final line of code returns the value of revString (return revString).

displayString the View Function

displayString's argument list:

  • revString

displayString, the final directive called in the controller function, is the view function of this short application. When invoked, the code blocked scoped within displays a brief message containing the reversed string, revString, the argument passed being the reversed string inserted into the brief message assigned its value by the innerHTML property of #msg. #msg is a paragraph HTML element in which its parent element is #alert, a division HTML element. In the final line of code, #alert's classList property is accessed and its remove method is called to which in effect removes the element's CSS visibility property.
